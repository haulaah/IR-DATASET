Incident Response Process Log Dataset

Welcome to exploring our synthetic Incident Response (IR) Process activities dataset generated using an LLM. It is made up of a set of synchronised application and communication logs from a set of linked tools typically deployed in SOC or CSIRT: i.e. SIEM, Ticketing System, Case Management System, Threat Intelligence Tools, and Observables Analyser.

Each IR case instance in the dataset is structured to represent a complete IR process (see below for a sample visualisation of a case).

Please review the visualisation, dataset and provide feedback on its accuracy in depicting realistic IR scenarios, as well as its completeness and consistency. Once you've completed your review, please fill out the feedback survey.

Dataset Structure

  • Types of Incidents: different Incident types, such as phishing attacks, brute force attacks, malware infection, DDoS etc.

  • Data Fields: the fields cover information on Incident data, analysis methods, timelines, classification, response actions, and communication data.

  • Number of Cases: The dataset comprises of 8 cases, each representing a unique IR scenario.
